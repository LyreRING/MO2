## 学习总结：

#### Elements：
*（fn+）f12打开，左上方处。
* 页面中间显示html源码，高亮的为当前选中的html elment。
* 在选中的html element对应的CSS属性窗口，可以进行修改，添加，选定属性是否被计算等。
* 左上角按钮可以快速选定element，预览手机端效果。
#### 其他界面：
* console：可以输入命令，输出中间变量进行debug。
* source：显示网站所有源文件及结构。
#### network面板：
* 包括控制器、过滤器、概览、请求列表、概要。
* 控制器——控制面板的外观与功能。
* 过滤器——过滤请求列表中显示的资源,按住Command（Mac）或Ctrl（Window/Linux），然后点击过滤器可以同时选择多个过滤器。
* 比较重要的xhr全称xml http request，实际上大部分类型的api请求，即使返回值不是xml（比如json）也会被分类在这一栏里。
* 概览——显示HTTP请求、响应的时间轴。
* 请求列表：</br>
	默认时间排序，可选择显示列。</br>
	查看所有的资源请求，包括网络请求，图片资源，html、css、js文件等请求。</br>
	可以根据需求筛选请求项，一般多用于网络请求的查看和分析，分析后端接口是否正确传输，获取的数据是否准确，请求头，请求参数的查看。</br>
* 概要——请求总数、总数据量、总花费时间等。
#### network请求列表：
* Name：资源的名称。
* Status：HTTP状态代码。
* Type：请求的资源的MIME类型。
* Initiator：发起请求的对象或进程，它可能有以下几种值：</br>
	Parser（解析器）：Chrome的 HTML 解析器发起了请求（鼠标悬停显示JS脚本）。</br>
	Redirect（重定向）：HTTP重定向启动了请求。</br>
	Script（脚本）：脚本启动了请求。</br>
* Other（其他）：一些其他进程或动作发起请求，例如用户点击链接跳转到页面或在地址栏中输入网址。</br>
* Size：服务器返回的响应大小（包括头部和包体），可显示解压后大小。
* Time：总持续时间，从请求的开始到接受响应中的最后一个字节。
* Waterfall：各请求相关活动的直观分析图。
#### network预览请求内容：
* 可查看头部，包括请求头、响应头。查看Cookie，预览响应正文：查看图像用等。
* Headers：请求方信息和请求、回复的header的集合，分为四块：</br>
	General：包含了请求的链接，请求的方法（post/get等，亦称为http谓词），服务器端回复的状态码，远程ip地址等。</br>
	Response Headers：服务器端回复里带的http header，里边比较重要的有set-cookie，这个header和身份认证相关。</br>
	Request Headers：浏览器发送的请求所带的http header。</br>
	Request Payload：请求信息的“body”，一般是我们传递给服务器的参数。有些请求没有参数，或全在url中，有些请求不允许拥有body（比如get）。</br>
* Preview：服务器的返回信息的预览。开发者可以通过这个窗口的信息在一定程度上判断后端是否正常工作。
* Response：服务器的回复信息，Preview窗口的源码。
* 开发时尽量打开浏览器匿名模式，这样浏览器在关闭后不会记忆你的cookie。
* GET：用于获取页面的指定信息（以实体格式）。若请求资源为动态脚本（非HTML），则返回文本是WEB容器解析后的HTML源代码，而不是源文件。
* POST：与GET方法相似，但POST有请求内容，多用于向服务器发送大量的数据，GET发送数据时大小（长度）有限制，且GET请求会将发送的数据显示在浏览器端，而POST不会，所以安全性相对较高。
